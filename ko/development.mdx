<Info>
  **전제 조건**: 계속하기 전에 Node.js(버전 19 이상)를 설치하세요. <br />
  계속하기 전에 `docs.json`로 업그레이드하고 레거시 `mint.json` 파일을 삭제하세요.
</Info>

운영 체제에 Mintlify를 설치하고 실행하려면 다음 단계를 따르세요:

**1단계**: Mintlify 설치:

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

**2단계**: docs 디렉토리(`docs.json` 파일이 있는 위치)로 이동하여 다음 명령을 실행하세요:

```bash
mintlify dev
```

문서의 로컬 미리보기는 `http://localhost:3000`에서 사용할 수 있습니다.

### 사용자 지정 포트

기본적으로 Mintlify는 포트 3000을 사용합니다. `--port` 플래그를 사용하여 Mintlify가 실행되는 포트를 사용자 지정할 수 있습니다. 예를 들어, Mintlify를 포트 3333에서 실행하려면 다음 명령을 사용하세요:

```bash
mintlify dev --port 3333
```

이미 사용 중인 포트에서 Mintlify를 실행하려고 하면 다음으로 사용 가능한 포트를 사용합니다:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify 버전

각 CLI 릴리스는 특정 버전의 Mintlify와 연결되어 있습니다. 로컬 웹사이트가 프로덕션 버전과 일치하지 않는 경우 CLI를 업데이트하세요:

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

## 링크 검증

CLI는 문서에서 만든 참조 링크를 검증하는 데 도움이 될 수 있습니다. 깨진 링크를 식별하려면 다음 명령을 사용하세요:

```bash
mintlify broken-links
```

## 코드 포맷팅

IDE에서 MDX를 인식하고 포맷팅하기 위한 확장 프로그램을 사용하는 것이 좋습니다. VSCode 사용자라면 구문 강조를 위한 [MDX VSCode 확장 프로그램](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)과 코드 포맷팅을 위한 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)를 고려해 보세요.

## 문제 해결

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    이는 오래된 버전의 node 때문일 수 있습니다. 다음을 시도해 보세요:

    1. 현재 설치된 mintlify 버전 제거: `npm remove -g mintlify`
    2. Node v19 이상으로 업그레이드하세요.
    3. mintlify 재설치: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    Solution: Go to the root of your device and delete the \~/.mintlify folder. Afterwards, run `mintlify dev` 다시 실행.
  </Accordion>
</AccordionGroup>

CLI 버전에서 무엇이 변경되었는지 궁금하신가요? [CLI 변경 로그를 확인하세요.](https://www.npmjs.com/package/mintlify?activeTab=versions)
