Weave Comparison 기능을 사용하면 코드, 트레이스, 프롬프트, 모델 및 모델 구성을 시각적으로 비교하고 차이점을 확인할 수 있습니다. 두 객체를 나란히 비교하거나 더 큰 객체 세트를 분석하여 차이점, 패턴 및 추세를 식별할 수 있습니다.

이 가이드는 비교를 시작하는 단계와 기준선 비교, 숫자 차이 형식 지정 등을 포함하여 비교 보기를 조정하는 데 사용할 수 있는 작업을 다룹니다.

## Comparison 보기 접근하기

1. 사이드바에서 비교하려는 객체 유형을 선택하세요(예: **Traces**, **Models** 등).
2. 비교하려는 객체를 선택하세요. 선택 방법은 비교하려는 객체 유형에 따라 다릅니다:
   * **Traces**의 경우, Traces 열의 해당 행에 있는 체크박스를 선택하여 비교할 트레이스를 선택합니다.
   * **Models**와 같은 객체의 경우, 모델 Versions 페이지로 이동하여 비교하려는 버전 옆의 체크박스를 선택합니다.
3. **Compare**를 클릭하여 Comparison 보기를 엽니다. 이제 [available actions](#available-actions)를 사용하여 보기를 세부 조정할 수 있습니다.

### 차이점 표시 변경

기본적으로 **Diff only**는 꺼져 있습니다. 변경된 행만 표시되도록 테이블 행을 필터링하려면 **Diff only**를 켜세요.

### 나란히 표시

> 이 옵션은 두 객체를 비교하거나 [pair from a multi-object comparison](#compare-a-pair-from-a-multi-object-comparison)할 때만 사용할 수 있습니다.

각 객체를 별도의 열에 나란히 비교하려면 **Side-by-side**를 선택하세요.

![Side-by-side Comparison view of two objects](imgs/comparison-2objs-sidebyside.png)

### 통합된 보기로 표시

> 이 옵션은 두 객체를 비교하거나 [pair from a multi-object comparison](#compare-a-pair-from-a-multi-object-comparison)할 때만 사용할 수 있습니다.

각 객체를 통합된 보기로 비교하려면 **Unified**를 선택하세요.

![Unified Comparison view of two objects](imgs/comparison-2objs-unified.png)

### 기준선 설정

기본적으로 Comparison 보기의 각 객체는 왼쪽에 있는 객체와 비교됩니다. 그러나 객체를 *baseline*으로 설정할 수 있으며, 이는 모든 객체가 보기의 가장 왼쪽 객체와 비교됨을 의미합니다.
객체를 기준선으로 설정하려면 다음을 수행하세요:

1. Comparison 보기 상단 바에서 기준선으로 설정하려는 객체 위에 마우스를 올립니다.
2. ID 오른쪽에 있는 세 개의 점을 클릭합니다.

   ![Make baseline option displayed.](imgs/comparison-2objs-baseline.png)
3. 드롭다운에서 **Make baseline**을 선택합니다. UI가 새로고침되어 기준선 객체가 상단 바의 가장 왼쪽에 위치하고 `Baseline`가 ID 옆에 표시됩니다.

   ![Baseline set.](imgs/comparison-2objs-baseline-set.png)

### 기준선 제거

객체를 기준선에서 제거하려면 다음을 수행하세요:

1. Comparison 보기 상단 바에서 기준선 객체 위에 마우스를 올립니다.
2. ID 오른쪽에 있는 세 개의 점을 클릭합니다.
3. 드롭다운에서 **Remove baseline**을 선택합니다. 이제 `Baseline`가 더 이상 호출 ID 옆에 표시되지 않습니다.

### 비교 순서 변경

비교 순서를 변경하려면 다음을 수행하세요:

1. Comparison 보기 상단 바에서 재정렬하려는 ID 위에 마우스를 올립니다.
2. ID 왼쪽에 있는 여섯 개의 점을 클릭합니다.

   ![Setting the order.](imgs/comparison-2objs-reorder.png)
3. 선택한 객체에 따라 ID를 왼쪽이나 오른쪽으로 드래그합니다.
4. ID를 원하는 순서에 배치합니다. UI가 업데이트된 비교 순서로 새로고침됩니다.

### 숫자 차이 표시 형식 변경

`completion_tokens`와 `total_tokens`와 같은 숫자 값의 경우, 차이를 정수 또는 백분율로 볼 수 있습니다. 또한 양수 값은 승수로 볼 수 있습니다. 숫자 차이의 표시 형식을 변경하려면 다음을 수행하세요:

1. Comparison 테이블에서 차이 표시 형식을 업데이트하려는 숫자 값을 찾습니다.

   ![A numeric value displayed as an integer.](imgs/comparison-2objs-numericdiffformat.png)
2. 차이 값을 클릭합니다. 형식이 자동으로 정수 또는 백분율로 업데이트됩니다.

   ![A numeric value updated to a percentage.](imgs/comparison-2objs-numericdiffformat-updated.png)

### 기준선 또는 이전과 비교

> 이 옵션은 3개 이상의 객체를 비교할 때만 사용할 수 있습니다.
> 또한 [set](#set-a-baseline)하거나 [remove an existing baseline by clicking the 3 dots to the right of the ID](#remove-a-baseline)할 수 있습니다.

3개 이상의 객체로 기준선 비교를 수행하려면 다음을 수행하세요:

1. Comparison 보기의 오른쪽 모서리에서 드롭다운을 클릭합니다. 현재 보기 구성에 따라 드롭다운 제목은 **Compare with previous**또는 **Compare with baseline**입니다.
2. 현재 보기 구성에 따라 **Compare with previous**또는 **Compare with baseline**을 선택합니다.
   * **Compare with baseline**: 가장 왼쪽 객체를 기준선으로 설정합니다. 테이블이 업데이트되어 가장 왼쪽 열이 기준선이 됩니다.
   * **Compare with previous**: 기준선으로 설정된 객체가 없습니다.

### 다중 객체 비교에서 쌍 비교

> 이 옵션은 3개 이상의 객체를 비교할 때만 사용할 수 있습니다.

3개 이상의 객체를 비교할 때 단일 객체를 이전 객체나 기준선과 비교할 수 있습니다. 이렇게 하면 Comparison 테이블 보기가 변경되어 두 객체 비교와 동일한 보기가 됩니다. 다중 객체 비교에서 객체 쌍을 비교하려면 다음을 수행하세요:

1. Comparison 보기 상단 바에서 이전 또는 기준선과 비교하려는 ID를 찾습니다.
2. 항목을 선택하려면 ID를 클릭합니다. UI가 두 방향 비교 테이블로 새로고침됩니다.

   ![Comparing a pair from a multi-object comparison.](imgs/comparsion-7objs-diffonly-subset.png)

비교를 위해 선택한 처음 6개 객체가 테이블에 표시되도록 보기를 재설정하려면 ID를 다시 클릭합니다.

### 비교에서 객체 제거

> 이 옵션은 3개 이상의 객체를 비교할 때만 사용할 수 있습니다.

비교에서 객체를 제거하려면 다음을 수행하세요:

1. 비교 보기 상단 바에서 비교에서 제거하려는 객체를 찾습니다.
2. ID 오른쪽에 있는 세 개의 점을 클릭합니다.
3. 드롭다운에서 선택하세요 **비교에서 객체 제거**. UI가 제거된 객체가 더 이상 포함되지 않은 업데이트된 테이블로 새로고침됩니다.

## 사용 참고사항

* 비교 기능은 UI에서만 사용할 수 있습니다.
* 원하는 만큼 많은 객체를 비교할 수 있습니다. 그러나 UI는 최대 6개만 표시합니다. 6개 이상의 객체를 비교할 때 비교 테이블에서 보이지 않는 객체를 보려면 [비교 순서 변경](#change-the-comparison-order)하여 해당 객체가 왼쪽에서 오른쪽으로 처음 6개 객체 중 하나가 되도록 하거나 [다중 객체 비교에서 쌍 비교](#compare-a-pair-from-a-multi-object-comparison)를 통해 쉽게 볼 수 있습니다.
